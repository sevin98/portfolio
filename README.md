# Portfolio Website

개인 포트폴리오 웹사이트 프로젝트

🌐 **사이트 링크**: [https://sevin.dev](https://sevin.dev)

## 📋 프로젝트 개요

Next.js 기반의 개인 포트폴리오 웹사이트로, 백엔드 개발자로서의 경험, 프로젝트, 기술 스택을 소개합니다.

## 🛠 기술 스택

- **Framework**: Next.js 16.1.1
- **Language**: TypeScript 5
- **Styling**: Tailwind CSS 4
- **UI Library**: React 19.2.3
- **Internationalization**: next-intl 4.7.0
- **Theme**: next-themes (다크모드 지원)
- **Markdown**: react-markdown, remark-gfm

- ✅ **영어 (en)** - English
- ✅ **중국어 (zh)** - Chinese

## 🔄 자동 버전 관리 시스템

이 프로젝트는 커밋할 때마다 자동으로 버전과 README를 업데이트합니다.

### 버전 증가 규칙

- **feat**: 마이너 버전 증가 (1.0.0 → 1.1.0)
- **fix, refactor, perf, style, docs, chore, test**: 패치 버전 증가 (1.0.0 → 1.0.1)
- **첫 배포**: 0.0.1 → 1.0.0으로 자동 설정

### 사용 방법

1. **Git Hook 설정** (최초 1회만 실행)
   ```bash
   npm run setup-hook
   ```

2. **커밋 시 자동 업데이트**
   - 커밋 메시지를 컨벤션에 맞게 작성 (예: `feat: 새로운 기능 추가`)
   - 커밋 후 자동으로:
     - `package.json`의 버전이 업데이트됩니다
     - `README.md`에 커밋 내역이 추가됩니다
   - 업데이트된 파일들은 자동으로 staging에 추가되므로, 다음 커밋에 포함시키려면:
     ```bash
     git add README.md package.json
     git commit -m "chore: update README"
     ```

3. **수동 업데이트** (필요시)
   ```bash
   npm run update-readme
   ```

### 커밋 메시지 컨벤션

- `feat: 새로운 기능 추가`
- `fix: 버그 수정`
- `refactor: 코드 리팩토링`
- `perf: 성능 개선`
- `style: 스타일 변경`
- `docs: 문서 수정`
- `chore: 기타 작업`

```
├── components/            # React 컴포넌트
│   ├── layout/           # 레이아웃 컴포넌트
│   ├── sections/         # 페이지 섹션
│   ├── ui/               # UI 컴포넌트
│   └── providers/        # Context Providers
├── data/                 # 데이터 파일
│   └── portfolio.ts      # 포트폴리오 데이터
├── lib/                  # 유틸리티 함수
├── messages/             # 다국어 메시지
│   ├── ko.json          # 한국어
│   └── en.json          # 영어
├── public/               # 정적 파일
└── i18n.ts              # 다국어 설정
```
